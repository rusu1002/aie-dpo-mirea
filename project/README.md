- **Краткое описание (2-4 предложения):**  
  `Проект посвящён разработке RAG-системы для ответа на вопросы пользователей по финансовой документации. Система использует гибридный поиск, объединяющий плотный (Dense Retrieval на эмбеддингах) и разреженный (BM25) ретривинг, а также реранжирование найденных документов для повышения качества ответов. ## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - `01_EDA.ipynb` – EDA, 
  - `02_baselines.ipynb` – бейзлайны, прототипы моделей, исследование методов.
- `src/` – основной код проекта:
  - `data/` – модули для работы с данными:
    - `chunking.py` – разбиение на чанки;
    - `loader.py` – загрузка данных;
    - `preprocessing.py` – предобработка данных;
  - `generation/` – обучение моделей:
    - `generator.py` – генерация ответа через api LLM или fallback из контекста;
    - `manager.py` – проверка доступности LMM через API, выбор модели;
  - `retriever/` – запуск сервиса/скриптов:
    - `bm25.py` – инициализация BM25 ретривера;
    - `fusion.py` – ретривинг чанков;
    - `reranker.py` – реранжирование чанков;
    - `vectorstore.py` – загрузка эмбеддера, создание эмбеддингов;
  - `service/` – запуск сервиса;
  - `utils/`:
    - `config.py` – необходимые константы;
    - `logger.py` – логирование.
- `configs/` – конфигурационные файлы:
  - `.env.example`
- `tests/` – тесты (юнит-тесты, простые проверки).
- `artifacts/` – сохранённые модели, отчёты, результаты экспериментов.

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.10`.

### 3.2. Установка окружения

Примерный сценарий (можно адаптировать под ваш проект):

```bash
# Перейти в папку проекта
cd project

# Создать виртуальное окружение (опционально, но рекомендуется)
python -m venv .venv

# Активировать окружение:
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate

# Установить зависимости
pip install --upgrade pip
pip install -r requirements.txt
```

Для работы системы необходимо создать файл .env на основе шаблона:
```bash
cp .env.example .env
```
или на Windows:
```bash
copy .env.example .env
```
В файле необходимо указать:
```bash
LLM_API_KEY=<ключ для доступа к LLM API>
RERANKER_API_KEY=<ключ для доступа к reranker API>
```

## 4. Как запустить проект

### 4.1. Запуск индексации

```bash
cd project
python -m src.train
```

### 4.2. Запуск сервиса

```bash
cd project
uvicorn src.service.service:app --reload
```

Опишите:

- После запуска сервис будет доступен по адресу: http://127.0.0.1:8000;
- Эндпоинты:
  - GET /health: Проверка работоспособности сервиса.
  - POST /predict: Основной эндпоинт для получения ответа RAG.
  - GET /docs: Автоматическая Swagger-документация FastAPI.
- как протестировать работоспособность: через эндпоинт /health:
```bash
curl http://localhost:8000/health
```

## 8. Ограничения и дальнейшая работа

Кратко опишите:

- В рамках проекта была реализована базовая RAG-система для ответа на финансовые вопросы пользователей, однако некоторые возможности не вошли в текущую версию;
- Основные ограничения:
  - сервис разворачивается локально и не контейнеризирован с помощью Docker;
  - не реализовано хранение истории запросов и ответов;
  - генерация выполняется через внешний LLM API, поэтому качество ответов зависит от доступности и характеристик выбранной модели;
  - эксперименты проведены преимущественно для ретривинг части, без детального сравнения различных генеративных моделей;

- Проект может быть расширен в нескольких направлениях:
  - добавить Docker-контейнеризацию;
  - организовать хранение истории запросов и ответов в базе данных;
  - провести сравнительный анализ различных LLM-моделей для генерации ответов;
  - протестировать несколько реранкеров и оценить их влияние на качество поиска;
  - добавить автоматическую оценку качества ответов с использованием Retrieval Metrics и LLM-as-a-Judge подходов;
  - реализовать автоматическое обновление индексов;
  - добавить веб-интерфейс для взаимодействия с системой;
  - расширить набор экспериментов и провести более детальное исследование влияния параметров чанкинга, эмбеддеров и методов гибридного поиска на качество ответов.
